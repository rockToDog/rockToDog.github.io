elementUI中碰到的一些问题

发表于 2017-10-31

一点经验

###项目
来到新公司不久，也是新起的项目，一个后台管理平台，我来的时候已经基本成形了，基本就是套了下vue-admin，也是第一次正式在工作上面使用vue，确实很简洁，但是写了大概几天，感觉还是有些随意，水平高的，经验丰富的选手能用的很好，经验稍微欠缺一点的，如果项目稍微复杂一点，可能就比较难看了
项目UI框架用的是elementUI，还是很好用的，但是比较坑的是，我也不知道为什么起项的时候2.0版本都已经出来了，但是项目上面用的居然还是1.1版本的，所以就陆陆续续的有一些坑出来了

###datepicker的问题
测试小妹妹说在选择日期的时候，如果是2月的话，第一次选中后，显示选中日期的输入框里面显示的3月的同一天，略微有些懵逼，今天看了一下午，最后确定是JS中日期方法setMonth的问题，具体代码如下
在elmentUI的datepicker包里面，找到date.vue文件，里面的handleDatePick方法有一段对this.date的设置

断点打到这个地方，等setMonth完了之后，打印出this.date.getMonth()的值，是2，但是明明value.getMonth()获取到的值是1，为啥设置的set的是1，get到的变成2了呢？MDN上有关于setMonth方法的说明

The current day of month will have an impact on the behaviour of this method. Conceptually it will add the number of days given by the current day of the month to the 1st day of the new month specified as the parameter, to return the new date. For example, if the current value is 31st August 2016, calling setMonth with a value of 1 will return 2nd March 2016. This is because in 2016 February had 29 days.
而且下面还贴心的给出了例子。。。

大概意思就是说如果现在是31号，设置月份为2月，日期就变成了2月31号，这明显是不对的，是因为2月只有29天，所以就到了3月份；

所以直接修改为this.date.setMonth(value.getMonth(), 1)，试了下，没问题了

回家后看了下1.4.6版本的代码，这个地方果然已经修改成了
